
Threa 란?

process : 프로그램이 실행되면 OS로 부터 메모리를 할당 받아서 프로세스 상태가 된다
프로세스 안에는 하나의 작업에 단위인 여러 Thread 를 가질 수 있다
thread : 하나의 프로세스 안에 실제 작업을 수행하는 하나의단이다

multi threading : 쓰레드가 여러개
여러 thread가 수행되는 프로그래밍, 여러 작업이 동시에 실행되는 효과를 얻을 수 있다
thread 는 각각에 자신만에 작업 공간을 가진다 (conrext)

각 thread 사이에 공유하는 자원이 있을 수 있다(코딩에 따라서)
여러 thread가 자원을 송유하는작업이 수핸되는경우 서로 자원을 차지 하려는
race condition 이 발생할 수 있다(--> 의도하지 않은 결과를 생성할 수 있다)

이렇게 여러 thread 가 공우하는 자원중 경쟁이 발생하는 부분을 critical section 이라고 한다
critical section 을 해결하기 다루기 위해서 동기화 처리를 하는 것이 바람직하다
(동기화 처리는 일종에 순차적 수행을 말한다)
